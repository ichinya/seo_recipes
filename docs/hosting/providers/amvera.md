# Amvera Cloud

[Amvera Cloud](https://amvera.ru/) - не классический VPS, а PaaS-облако для запуска приложений, сайтов и сервисов без самостоятельной настройки виртуальных машин.

Панель проектов находится на [cloud.amvera.ru](https://cloud.amvera.ru/projects), но без авторизации открывается только вход в аккаунт. Для публичной информации лучше смотреть основной сайт и документацию.

## Что заявлено на сайте

- деплой через `git push` или загрузку файлов из интерфейса;
- поддержка Python, Java, Node.js, Go, C#, Ruby, PHP, Docker и других окружений;
- managed-кластеры PostgreSQL, Redis, MongoDB и MySQL;
- регионы доступности в Москве и Варшаве;
- пользовательские данные и репозитории хранятся в трех экземплярах;
- ежедневный бэкап в независимом дата-центре;
- автоматический перезапуск проектов при сбоях;
- стартовый баланс 111 рублей на тест.

## Плюсы

- подходит для MVP, ботов, небольших сайтов и сервисов, где не хочется администрировать сервер;
- проще классического VPS, если проект нормально упакован и не требует ручной настройки ОС;
- есть GitOps-подход и Docker;
- есть российское юрлицо и публичные реквизиты.

## Минусы и риски

- это не VPS: меньше контроля над системой и сетевым окружением;
- нужно проверять, подходит ли модель деплоя под конкретный стек;
- для нестандартных задач может быть сложнее, чем обычный сервер;
- тарифы и лимиты нужно смотреть перед запуском конкретного проекта.

## Итог

Amvera Cloud стоит держать отдельно от VPS-провайдеров: это PaaS для быстрого деплоя приложений. Хороший кандидат на тест для небольших проектов, где важнее скорость запуска, чем полный контроль над сервером.
