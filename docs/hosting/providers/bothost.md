# BotHost

BotHost (bothost.ru) - не классический VPS, а специализированная PaaS-платформа для запуска ботов без ручной настройки сервера, systemd, SSL и базовой инфраструктуры. Позиционируется как хостинг ботов для мессенджеров: загрузка кода, автоматическое развертывание в Docker-контейнере и мониторинг из панели.

Формат удобен для Telegram-ботов, пет-проектов и небольших сервисов, когда важнее быстро запустить бота, чем администрировать сервер.

## Что заявлено

- хостинг ботов для Telegram, Discord, VK, WhatsApp, Slack и Twitch;
- поддержка Python, Node.js и PHP;
- автоматическое развертывание и запуск в Docker-контейнерах;
- логи и мониторинг из панели;
- webhook-интеграция;
- привязка через Telegram;
- заявленный аптайм 99.9%;
- бесплатный тариф для тестирования.

## Плюсы

- заточен именно под ботов мессенджеров, а не под универсальный VPS;
- не нужно вручную настраивать сервер, systemd, SSL и деплой;
- запуск в Docker-контейнерах;
- есть бесплатный тариф для теста и низкий порог входа (99 ₽/мес);
- поддержка Python, Node.js и PHP;
- логи, мониторинг и webhook из коробки;
- рублевая оплата.

## Минусы и риски

- это не VPS: root-доступа и полного контроля над окружением ждать не стоит;
- бесплатный тариф очень маленький: 1 бот, 256 МБ RAM, 0.5 ГБ диска и без env-переменных;
- заявленный аптайм 99.9% нужно проверять на практике и внешним мониторингом;
- бэкапы, поведение на долгих фоновых задачах, очередях и после падения процесса нужно проверять на реальном боте;
- условия оферты, возвраты и SLA нужно уточнять отдельно.

## Что тестировать

- зарегистрироваться и задеплоить минимального Telegram-бота на Python или Node.js;
- проверить хранение токенов и переменные окружения на платном тарифе;
- посмотреть логи, рестарт и обновление кода из панели;
- проверить webhook и работу под нагрузкой;
- измерить холодный старт и поведение после падения процесса;
- поставить внешний uptime-мониторинг на 7 дней;
- уточнить у поддержки условия бэкапов, возвратов и лимитов тарифов.

## Итог

BotHost выглядит как нишевый кандидат для хостинга ботов, когда нужно быстро запустить бота без DevOps-рутины. Это не замена VPS, а упрощенная платформа деплоя под мессенджеры.

Практический статус: добавить в список на тест. Для рабочих проектов сначала проверить лимиты дешевых тарифов, бэкапы, аптайм и поддержку на реальном боте.
