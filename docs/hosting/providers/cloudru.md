# Cloud.ru

[Cloud.ru](https://cloud.ru/partners/referral/form?referralCode=592010788096) - отдельный российский облачный провайдер. 

## Статус заметки

По текущему впечатлению Cloud.ru относится к более дорогим, но сильным по комплаенсу российским облакам. В заметках отдельно зафиксирован вариант с маркетинговыми условиями, где основная регулярная оплата может сводиться к публичному IP примерно за 150 рублей. Это нужно перепроверять перед использованием, потому что условия акций и тарификации меняются.

## Что известно

Cloud.ru развивает облачные и AI-сервисы для бизнеса. По официальной главной странице у провайдера больше 130 IaaS- и PaaS-сервисов, а в линейке есть виртуальные машины, хранилища, базы данных, Kubernetes, AI/ML и корпоративные инфраструктурные продукты.

Практический вывод:

- для простого дешевого VPS Cloud.ru может быть избыточным;
- для проектов с российской юрисдикцией, документами и требованиями ИБ это один из основных кандидатов;
- перед запуском нужно внимательно считать цену IP, дисков, снимков, трафика и managed-сервисов.

## Сертификация и безопасность

По официальной документации Cloud.ru:

- все платформы Cloud.ru имеют уровень защиты 152-ФЗ УЗ-1;
- облачная платформа для ГИС аттестована для государственных информационных систем с первым классом защищенности К1;
- платформа "Облако VMware" аттестована по требованиям безопасности информации для третьей категории значимости объектов КИИ по 187-ФЗ;
- реализованы ISO/IEC 27001, ISO/IEC 27701, ISO/IEC 27017 и ISO/IEC 27018;
- есть документы по требованиям 719-П ЦБ РФ и ГОСТ Р 57580.1-2017;
- зоны доступности платформ VMware и Advanced соответствуют PCI DSS 3.2.1, а для Advanced также указан сертификат PCI DSS 4.0;
- указаны лицензии Роскомнадзора, ФСБ и ФСТЭК.

Важная оговорка из документации: Cloud.ru не становится оператором персональных данных вместо клиента. Ответственность за доступы, настройки и корректную обработку данных остается на клиенте.

## Плюсы

- Очень сильная публичная база по сертификатам, аттестатам и лицензиям;
- подходит для проектов, где важны российская инфраструктура и документы по ИБ;
- есть широкий набор IaaS/PaaS-сервисов;
- есть варианты для ГИС, КИИ, PCI DSS и ГОСТ Р 57580;
- можно проверять маркетинговые условия, если нужен недорогой старт в российском облаке.

## Минусы и риски

- Не бюджетный VPS-провайдер;
- для небольших проектов может быть сложнее и дороже простого VDS;
- комплаенс не снимает обязанности с клиента: нужны правильные настройки, документы и процессы.

## Что тестировать перед рекомендацией

- реальную стоимость минимальной VM с IP, диском, бэкапами и трафиком;
- доступность нужных сервисов для физлица, ИП или юрлица;
- скорость поддержки до оплаты и после создания ресурсов;
- правила по SMTP, PTR, антиабузу и DDoS;
- применимость сертификатов к конкретной услуге и архитектуре.

## Итог

Cloud.ru стоит держать как отдельный вариант для серьезных российских проектов, где важны документы, ИБ и регуляторные требования. Для простого дешевого VPS лучше сначала смотреть более легкие провайдеры.
