# Deploy-F

Deploy-F - не классический VPS, а PaaS-хостинг для быстрого запуска приложений без ручной настройки сервера, доменов, SSL, systemd и базовой инфраструктуры. По рекламе сценарий такой: загружаешь ZIP-архив с кодом, и через несколько секунд проект уже работает в сети.

Это выглядит полезно для MVP, Telegram-ботов, пет-проектов и небольших сервисов, особенно если проект собран через AI и хочется быстро проверить идею без DevOps-рутины.

## Что пришло в рекламе

В рекламной заметке Deploy-F описан как хостинг для тех, кто хочет запускать продукты, а не администрировать серверы.

Заявленные возможности:

- деплой через ZIP-архив с кодом;
- поддержка Python, Node.js, Go и Docker;
- управление через Telegram-бота;
- автоперезапуск при сбоях;
- живой саппорт;
- бесплатный старт;
- платные тарифы от 300 рублей в месяц.

- в FAQ указана поддержка Python, Node.js, Go, PHP, Java, Ruby, Rust, .NET Core, статических приложений, Docker-контейнеров, Telegram-ботов и баз данных MySQL, PostgreSQL, MongoDB, MSSQL;
- управление заявлено через личный кабинет и Telegram-бота: логи, перезапуск приложения, обновление кода и очистка кэша;

Реальный проект пока не деплоился, поэтому это предварительная заметка по рекламе и открытым условиям, а не полноценный отзыв эксплуатации.

## Важные условия из оферты

У Deploy-F есть публичная оферта от ИП Погорельских Ксении Сергеевны, ИНН 631936108142. Расчеты идут в рублях, услуги предоставляются по предоплате.

Особенно важно:

- сервис стремится обеспечивать SLA не менее 99.0% в календарном месяце;
- компенсация при нарушении SLA - сервисный кредит, а не возврат денег;
- платежи за продление услуг не возвращаются;
- при оплате картой может использоваться автоматическое продление, его нужно отдельно отключать через поддержку;
- при DDoS-атаке, которая влияет на инфраструктуру провайдера, услугу могут приостановить, а возврат средств не производится;
- поддержку можно запросить через Telegram `@deployf_support`;
- оферта отдельно говорит, что поддержка не обязана консультировать по программированию, веб-дизайну, настройкам скриптов и похожим вопросам;
- клиент сам отвечает за резервные копии своих данных.

Последний пункт важен: маркетинг обещает помощь "с хостингом и кодом", но договорная обязанность поддержки по программированию ограничена. Это не минус сам по себе, но ожидания лучше держать реалистичными.

## Плюсы

- хороший формат для быстрого MVP или пет-проекта;
- не нужно вручную настраивать VPS, systemd, nginx, SSL и базовый деплой;
- есть ZIP-деплой, Git-деплой и Docker-сценарии;
- управление через Telegram-бота удобно для простых операций;
- есть автоперезапуск приложения;
- есть готовые базы данных MySQL, PostgreSQL, MongoDB и MSSQL;
- есть тариф от 300 рублей в месяц;
- российское юрлицо и публичная оферта;
- подходит для маленьких Telegram-ботов, API, демо и учебных проектов.

## Минусы и риски

- это не VPS: root-доступа и полного контроля над окружением ждать не стоит;
- самый дешевый тариф очень маленький по ресурсам: 0.1 CPU, 0.1 ГБ RAM и 0.4 ГБ диска;
- бесплатный старт нужно отдельно проверять в панели: публичная таблица тарифов показывает платный старт от 300 рублей;
- для production-проекта нужно проверить логи, секреты, переменные окружения, бекапы, метрики и лимиты;
- SLA 99.0% ниже, чем у многих зрелых облаков, и компенсация только сервисным кредитом;
- платежи за продление не возвращаются;
- при DDoS или нагрузке, которая мешает инфраструктуре, услугу могут приостановить;
- поддержка не обязана решать задачи программирования, несмотря на дружелюбный маркетинг;
- непонятно, как платформа поведет себя на долгих фоновых задачах, очередях, WebSocket, cron и stateful-сервисах без практического теста.

## Что тестировать

Перед рекомендацией стоит прогнать реальный небольшой проект:

- зарегистрироваться и проверить, есть ли бесплатный старт без карты;
- задеплоить минимальный Node.js/Python бот через ZIP;
- проверить деплой через Git и Docker;
- проверить переменные окружения и хранение токенов;
- посмотреть логи, рестарт, очистку кэша и обновление кода через Telegram-бота;
- проверить HTTPS, домен `deploy-f.com` и подключение своего домена;
- проверить базу PostgreSQL или MySQL на минимальном тарифе;
- измерить холодный старт, время перезапуска и поведение после падения процесса;
- проверить ограничения CPU/RAM/диска на дешевом тарифе;
- поставить внешний uptime-мониторинг на 7 дней;
- написать в поддержку с вопросом по бесплатному старту, лимитам и реальному объему помощи по коду;
- проверить, как быстро можно выгрузить проект, базу и секреты для переезда.

## Итог

Deploy-F выглядит как интересный кандидат для MVP, Telegram-ботов и пет-проектов, когда важнее быстро запустить код, чем получить полный контроль над сервером. Это не замена VPS, а упрощенная платформа деплоя.

Практический статус: добавить в список на тест. Для рабочих проектов сначала проверить бесплатный старт, лимиты дешевого тарифа, автоперезапуск, бекапы, логи, деплой через Docker/Git и качество поддержки на реальном кейсе.
