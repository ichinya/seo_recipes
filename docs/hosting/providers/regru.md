# REG.RU

REG.RU - отдельный хостинг и облачная экосистема. 

## Личный опыт

В REG.RU есть пара серверов. За примерно 15 лет по серверам не было необходимости писать в техническую поддержку.

По другим вопросам, например по партнерке и доменам, поддержка обычно отвечает быстро: иногда примерно в пределах 30 минут.

## Что известно

REG.RU предлагает VPS/VDS, виртуальный хостинг, домены, почту, SSL и связанные услуги. В продуктовой навигации часть облачной инфраструктуры уводит в Рег.облако.

По странице VPS у REG.RU отмечены:

- собственная платформа виртуализации;
- работа компании с 2006 года;
- русскоязычная база знаний;
- дата-центр в Москве для VPS на странице услуги;
- отдельные направления VPS, облачных VPS, выделенных серверов и управляемых услуг.

## Локации

По справке REG.RU физические серверы размещены в российских дата-центрах, основные локации:

- Москва;
- Санкт-Петербург;
- Тольятти.

Отдельная заметка на 11 июня 2026 года: было предположение, что у REG.RU появились локации вроде Узбекистана. По открытым страницам REG.RU и Рег.облака это пока не подтвердилось, поэтому Узбекистан не стоит записывать как факт.

## REG.RU и Рег.облако

Для 152-ФЗ важно разделять обычный REG.RU и Рег.облако.

По обычному хостингу и обычным серверным услугам REG.RU пока нет подтвержденной информации в этой заметке, что именно эти услуги имеют ФЗ-152 / УЗ-1.

У Рег.облака есть отдельное решение "Серверы ФЗ-152": защищенные облачные серверы для хранения и обработки персональных данных. В документации указано, что инфраструктура размещается в сертифицированном ЦОДе на территории РФ, а Рег.облако предоставляет сертифицированный ЦОД и комплекс технических мер. Оператором персональных данных при этом остается клиент.

## Быстрая проверка

Проверка 11 июня 2026 года из текущей сети:

- `https://www.reg.ru/vps/` открылся с HTTP 200 примерно за 0.5 секунды;
- справка REG.RU по физическому расположению серверов открылась с HTTP 200 примерно за 0.3 секунды;
- `https://reg.cloud/cloud/fz152` открылся с HTTP 200 примерно за 0.6 секунды.

Это только проверка публичных страниц. Новый сервер не создавался.

## Плюсы

- Есть личный длительный опыт без обращений в поддержку по серверам;
- поддержка по доменам и партнерским вопросам отвечала быстро;
- крупный и давно работающий российский провайдер;
- есть VPS, хостинг, домены и сопутствующие услуги в одной экосистеме;
- для 152-ФЗ есть отдельное направление Рег.облака.

## Минусы и риски

- Сертификацию нельзя автоматически переносить с Рег.облака на обычные услуги REG.RU;
- для 152-ФЗ нужно выбирать конкретный защищенный продукт и проверять документы;
- зарубежные локации вроде Узбекистана пока не подтверждены публичными страницами;
- цены и доступность тарифов нужно проверять перед запуском.

## Что тестировать перед рекомендацией

- актуальные тарифы VPS и реальную цену с IP, дисками и бэкапами;
- доступность нужной локации;
- поддержку по техническому вопросу именно для VPS;
- правила по SMTP, PTR, антиабузу и DDoS;
- документы по 152-ФЗ, если проект работает с персональными данными.

## Итог

REG.RU остается отдельным рабочим вариантом с длительным личным опытом без заметных серверных проблем. Но для проектов с персональными данными нужно не просто брать обычный VPS, а отдельно проверять защищенные услуги Рег.облака и договорные условия.
