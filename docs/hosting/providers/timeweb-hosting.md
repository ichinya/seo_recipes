# Timeweb-хостинг

[Timeweb](https://timeweb.com/ru/?i=120481) - отдельный обычный хостинг для сайтов. Его лучше не смешивать с [Timeweb Cloud](./timeweb-cloud.md): у Cloud своя инфраструктура, свои аварии и инциденты, а обычный хостинг по текущему личному опыту работает спокойно.

## Контекст использования

На Timeweb-хостинге размещено несколько сайтов. Проблем с ними не было: сайты работают нормально, без заметных простоев или обращений из-за аварий.

Это именно опыт обычного хостинга для сайтов, а не VPS, облачных баз данных или инфраструктурных сервисов Timeweb Cloud.

## Плюсы

- Стабильная работа сайтов;
- подходит для обычного размещения сайтов без самостоятельного администрирования VPS;
- по текущему опыту не было заметных проблем;
- можно рассматривать отдельно от инцидентов в Timeweb Cloud;
- есть изоляция сайтов друг от друга;
- для сайтов можно выбирать версии PHP и Python;
- хороший вариант, когда нужен не сервер, а простой хостинг для сайтов.

## Минусы и риски

- Это не VPS и не облако: меньше контроля над окружением;
- для сложных проектов с нестандартной конфигурацией может быть удобнее Cloud или отдельный сервер;
- тарифы, лимиты, доступные версии PHP, Python и баз данных нужно проверять перед переносом;
- не стоит переносить на обычный хостинг все выводы по Timeweb Cloud: это разные продукты.

## Итог

Обычный Timeweb-хостинг по текущему опыту работает отлично: несколько сайтов размещены без проблем и заметных простоев. Поэтому хостинг можно оценивать мягче, чем Cloud, и не переносить на него риски облачной платформы.
