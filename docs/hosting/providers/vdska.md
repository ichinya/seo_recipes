# VDSka

[VDSka](https://vdska.ru/?p=21021) - недорогой VPS-хостинг, который использовался для серверов в США.

## Контекст использования

В работе было 3 сервера, все брались в США. Раньше тарифы были примерно по 300 рублей, позже подорожали; сейчас минимальный тариф зависит от локации и стоит примерно 470-550 рублей в месяц. В тарифе есть 3 ТБ трафика, а все сверх этого оплачивается отдельно.

## Минимальный тариф

На момент проверки минимальная конфигурация VDSka выглядела так:

| Параметр | Значение |
| --- | --- |
| CPU | 1 vCPU |
| RAM | 1 ГБ |
| Диск | 15 ГБ NVMe |
| Порт | 1 Gbit/s |
| Трафик | 3 ТБ в месяц |
| Цена | от 470 до 550 рублей в месяц, зависит от локации |

## Тесты VDS

23 июня 2026 года были проверены два сервера VDSka в США: сначала Miami / `us2`, затем Dallas / `us1`. Оба сервера были в похожей минимальной конфигурации: 1 vCPU, 1 ГБ RAM, 15 ГБ NVMe, порт 1 Gbit/s и включенный трафик 3 ТБ.

### Miami / us2

Первый тест делался в локации Miami / `us2`. Конфигурация в панели: 1 vCPU, 1 ГБ RAM, 15 ГБ NVMe, включенный трафик 3 ТБ. На момент проверки в панели было использовано 0.85 ТБ из 3 ТБ.

Внутри сервера была Debian 13 с ядром `6.12.73+deb13-amd64`, KVM-виртуализация, 1 vCPU на `Intel Xeon Platinum 8160 CPU @ 2.10GHz`, 967 MiB RAM, 1 GiB swap и корневой диск 15G. Хотя в панели встречались обозначения `ru1` / `us2`, по маршрутам сервер выглядел как американская локация с выходом через Miami: до ближайших внешних узлов задержка около 1-3 мс, а до российских точек примерно 140-160 мс.

Быстрый прогон через публичный скрипт на базе списка [itdoginfo/russian-iperf3-servers](https://github.com/itdoginfo/russian-iperf3-servers) дал такую картину:

| Город | Download | Upload | Ping |
| --- | ---: | ---: | ---: |
| Moscow | 78.5 Mbps | 441.3 Mbps | 152 ms |
| Saint Petersburg | 39.8 Mbps | 507.0 Mbps | 142 ms |
| Nizhny Novgorod | 52.5 Mbps | 443.5 Mbps | 141 ms |
| Chelyabinsk | 52.7 Mbps | 486.4 Mbps | 159 ms |
| Tyumen | 47.4 Mbps | 513.3 Mbps | 164 ms |

Ручные `iperf3`-замеры до российских точек:

| Направление | Upload | Download `-R` | Комментарий |
| --- | ---: | ---: | --- |
| Nizhny Novgorod | 236 Mbps | 567 Mbps | много ретрансмитов, но канал заметно выше 100 Мбит/с |
| Tyumen | 159 Mbps | 579 Mbps | много ретрансмитов, высокая задержка до РФ |
| Moscow | нет маршрута | нет маршрута | `mskst.st.mtsws.net:3333` не открылся с ошибкой `No route to host` |

Дополнительный международный `iperf3`-прогон по списку [iperf.fr](https://iperf.fr/iperf-servers.php) для Miami дал смешанный результат. До France / Paris обратное направление `-R` прошло на уровне 648 Mbps receiver / 704 Mbps sender, но с большим числом ретрансмитов. Прямое направление до этой точки завершилось сбросом соединения, а одна из следующих публичных международных точек не ответила по таймауту.

| Последовательное чтение | 785 MiB/s |

### Dallas / us1

Второй тест делался в локации Dallas / `us1`. Конфигурация в панели: 1 vCPU, 1 ГБ RAM, 15 ГБ NVMe, включенный трафик 3 ТБ. На момент проверки в панели было использовано 0.24 ТБ из 3 ТБ. В панели локация отмечена как `us1`, а по трассировкам сервер идет через Dallas / DFW.

Внутри сервера была Debian 13 с ядром `6.12.74+deb13+1-amd64`, KVM-виртуализация, 1 vCPU на `Intel Xeon Platinum 8170 CPU @ 2.10GHz`, 967 MiB RAM, 1 GiB swap и корневой диск 15G. До ближайших внешних узлов задержка около 1-3 мс, до российских точек примерно 145-169 мс.

Быстрый прогон через публичный скрипт на базе списка [itdoginfo/russian-iperf3-servers](https://github.com/itdoginfo/russian-iperf3-servers):

| Город | Download | Upload | Ping |
| --- | ---: | ---: | ---: |
| Moscow | 71.4 Mbps | 520.4 Mbps | 147 ms |
| Saint Petersburg | 32.9 Mbps | 272.1 Mbps | 162 ms |
| Nizhny Novgorod | 48.5 Mbps | 407.4 Mbps | 145 ms |
| Chelyabinsk | 57.5 Mbps | 396.7 Mbps | 166 ms |
| Tyumen | 59.7 Mbps | 475.1 Mbps | 169 ms |

Ручные `iperf3`-замеры до российских точек:

| Направление | Upload | Download `-R` | Комментарий |
| --- | ---: | ---: | --- |
| Nizhny Novgorod | 359 Mbps | 685 Mbps | много ретрансмитов, но результат выше, чем у Miami |
| Tyumen | 160 Mbps | 604 Mbps | близко к Miami по upload, немного выше по download |
| Moscow | нет маршрута | нет маршрута | `mskst.st.mtsws.net:3333` не открылся с ошибкой `No route to host` |

### Вывод по тестам

Для дешевого VPS в США оба результата нормальные. Miami и Dallas дают близкую задержку до России: примерно 140-170 мс, поэтому это не вариант для задач, где нужен низкий ping до РФ. Сеть до России асимметричная: в публичном тесте download из российских точек обычно около 33-79 Mbps, upload около 272-520 Mbps. Ручные `iperf3`-замеры показывают более высокий download через `-R`, но с большим числом ретрансмитов. Международный тест Miami до France / Paris тоже показал высокий `-R`, но стабильность публичных точек и маршрута нужно перепроверять.

Dallas в этом тесте выглядит лучше по диску, CPU и памяти: последовательная запись 1543 MiB/s против 1197 MiB/s у Miami, чтение 1004 MiB/s против 785 MiB/s, случайные операции тоже выше. Для фоновых сервисов, ботов, небольших сайтов, прокси-задач и сценариев, где нужна именно США-локация, VDSka выглядит рабочим бюджетным вариантом. Перед важной нагрузкой стоит повторить тест в другое время суток и проверить нужные маршруты.

## Плюсы

- невысокая цена даже после подорожания;
- есть включенный объем трафика;
- техническая поддержка после переключения на человека отвечала быстро;
- серверы подходят для задач, где нужна локация США;
- на тестовых серверах диск показал хорошие результаты для бюджетного VPS.

## Минусы и риски

- в начале обращения отвечает ИИ, и по опыту он плохо помогает;
- трафик сверх включенного лимита оплачивается отдельно;
- для пользователей из России задержка до российских точек высокая, примерно 140-160 мс;
- сеть до РФ асимметричная: входящий трафик из российских точек заметно слабее исходящего;
- цены и лимиты нужно перепроверять перед заказом.

## Итог

VDSka выглядит как нормальный бюджетный вариант для VPS в США. Главный раздражающий момент - первый уровень поддержки через ИИ, но после перехода к человеку ответы были быстрыми.

После тестов 23 июня 2026 вывод стал конкретнее: это не вариант для низкой задержки до России, зато для дешевого сервера в США с 3 ТБ трафика и быстрым диском выглядит нормально. Из двух проверенных площадок Dallas в этом прогоне оказался сильнее Miami по диску, CPU и памяти.
